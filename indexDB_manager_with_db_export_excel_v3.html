Best Practices for Multi-App Architecture

Enterprise Standard Operating Procedure (SOP)

1. The Origin Boundary

IndexedDB is scoped to the Origin (Protocol + Domain + Port). If you run multiple apps on https://intranet.company.com/app1 and https://intranet.company.com/app2, they technically share the same database storage space.

Risk: Without naming conventions, App 1 could overwrite App 2's data if they both use a generic name like "UserDB".

2. Naming Conventions

To ensure isolation, adopt a strict prefixing strategy for your database names:

  • Structure: [APP_CODE]_[ENVIRONMENT]_[DB_NAME]
  • Example 1: HR_PROD_Employees
  • Example 2: FIN_DEV_Invoices

3. Managing Connections

When updating database versions (schema changes), open connections in other tabs can block the upgrade. Implement the onversionchange handler in your applications to automatically close connections when a newer version attempts to load. 

db.onversionchange = function() {
    db.close();
    alert("Database is outdated, please reload the page.");
};
